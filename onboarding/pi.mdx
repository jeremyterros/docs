## Import des convives

Serenest recoit un fichier contenant :

- Nom
- Prenom
- Email
- Badge

Ce fichier est integre dans les systemes PI. Un email de masse est envoye aux convives pour qu'ils puissent installer l'application et s'inscrire en remplissant les champs requis (nom, prenom, email, badge).

## Verification et correspondance avec PI

Pour chaque inscription, l'application compare les donnees saisies par l'utilisateur avec les donnees existantes dans PI :

- **Le badge correspond au site indique** : inscription validee, compte lie a PI.
- **Le badge ne correspond pas au site** :
  - Inscription bloquee
  - Pop-up demandant a l'utilisateur de contacter le responsable du site

### Email

Utilise pour l'authentification et la communication (notifications, reinitialisation du mot de passe, informations convives).

### Logique de badge

Lorsqu'un utilisateur tente de se connecter avec un **badge de site**, le systeme applique la logique suivante :

| Scenario | Comportement |
| --- | --- |
| Aucun compte associe au badge | Un **nouveau compte est automatiquement cree** |
| Compte existant sans email | L'email saisi est **attribue a ce compte** qui devient le compte principal |
| Compte existant avec un email different | Une **erreur est retournee**, indiquant que le badge est deja associe a un autre compte |

## Processus d'inscription

### Avec SSO

**Utilisateur deja dans Serenest :**
- Connexion avec le compte existant
- Mise a jour des informations de site et d'entreprise
- Acces refuse si le role est *manager* ou *admin*
- Prenom et nom mis a jour
- Mot de passe reinitialise (non necessaire avec le SSO)

**Nouvel utilisateur Serenest :**
- Creation de compte avec les informations du SSO (prenom, nom, site, entreprise)

**Correspondance avec PI :**

| Donnees disponibles | Comportement |
| --- | --- |
| Avec email | Recherche du compte PI et mise a jour du badge |
| Avec badge | Recherche du compte PI par badge et recuperation de l'email |
| SSO avec email et badge | Si l'email n'existe pas, creation d'un nouveau compte (sans verifier le badge). Si l'email existe, verification du site et de l'identite de l'utilisateur |
| Sans email | Recuperation via le badge (protection = identifiant SSO du site — UUID) |
| Sans email et sans badge | Impossible, le compte n'existe pas dans PI |
| Sans badge | Impossible, tous les comptes PI ont un badge |

<Info>
  L'adresse email est utilisee cote Serenest pour creer le compte, se connecter et verifier si le compte existe deja.
</Info>

### Sans SSO

**Inscription manuelle :**
- L'utilisateur clique sur *Creer un compte*
- Verification que le badge correspond au site indique
- L'utilisateur est automatiquement lie a l'entreprise correspondant au site
