## Statistiques et facturation

- Côté Innovorder : créer un compte de service API
- Envoyer à Serenest :
  - **Brand ID** du nouveau site
  - **JWT Secret**
  - **Webhook Secret Key**
- Serenest configure le Webhook dans les intégrations du back-office Innovorder
- Serenest peut accéder directement à la plateforme IAS et créer le nouveau site en ajoutant le Brand ID et le JWT Secret. Cela permet à la plateforme de se connecter directement aux données Innovorder. Aucune action supplémentaire n'est requise concernant les statistiques et la facturation.

## Application mobile

<Steps>
  <Step title="Créer le site">
    Créer le site dans IAS ou le Back-Office Serenest (s'il est créé sur une plateforme, il sera automatiquement créé sur l'autre).
  </Step>
  <Step title="Créer un utilisateur de test">
    Créer un utilisateur et un badge de test pour le nouveau site.
  </Step>
  <Step title="Tester l'inscription">
    Tester l'inscription via ce badge dans l'application mobile, en s'assurant de sélectionner le bon site.
  </Step>
  <Step title="Tester le rechargement">
    Tester le rechargement du badge.
  </Step>
</Steps>
