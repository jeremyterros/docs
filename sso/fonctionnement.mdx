## Description générale

Le SSO simplifie l'expérience utilisateur. Une fois connecté à l'application principale de l'entreprise, les utilisateurs peuvent accéder à Serenest sans nouvelle authentification.

## Acteurs clés

| Acteur | Rôle |
| --- | --- |
| **Fournisseur d'identité (IdP)** | L'application source où l'utilisateur est déjà authentifié |
| **Fournisseur de service (SP)** | Serenest |

## Configuration par site

Chaque site dans Serenest est identifié par un **UUID unique** et dispose de sa propre configuration SAML indépendante. Cela permet à plusieurs entreprises de connecter leur propre fournisseur d'identité sur un même environnement Serenest.

## Flux SAML

<Steps>
  <Step title="Clic SSO">
    L'utilisateur clique sur un bouton SSO dans l'application tierce.
  </Step>
  <Step title="Authentification par l'IdP">
    Le fournisseur d'identité authentifie l'utilisateur et génère une assertion SAML signée.
  </Step>
  <Step title="POST de la SAMLResponse">
    L'IdP envoie la SAMLResponse (XML Base64) directement vers l'ACS URL de Serenest via le navigateur de l'utilisateur.
  </Step>
  <Step title="Validation et provisionnement">
    Serenest vérifie la signature, extrait les attributs utilisateur, crée ou met à jour le compte, puis génère un token d'authentification.
  </Step>
  <Step title="Connexion automatique">
    Le cookie de session est posé et l'utilisateur est redirigé dans l'application Webviews Serenest.
  </Step>
</Steps>
