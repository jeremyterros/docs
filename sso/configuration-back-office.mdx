## Rechargement de badge

Deux options de fournisseur : **Adoria** ou **Pi Electronique (PiPay)**.

### Paramètres requis

- **URL de rechargement** : spécifique à Adoria ou Pi
- **URL racine Pi Electronique**
- **Clé PiPay** : utilisée pour sécuriser les transactions

### Étapes de configuration

<Steps>
  <Step title="Saisir l'URL et la clé">
    Entrer l'URL et la clé dans le back-office.
  </Step>
  <Step title="Vérifier la validité">
    Vérifier la validité de la clé auprès du fournisseur.
  </Step>
  <Step title="Tester le rechargement">
    Effectuer des tests de rechargement de badge avec un compte de test.
  </Step>
</Steps>

### Notes spécifiques pour PiPay

- Une transaction avec un montant de `0` inscrit l'utilisateur au service PiPay (automatique lors de la première connexion).
- Endpoint impliqué : `envoi_epaiement_compte`.

## Accès aux Webviews

**Objectif** : intégrer Serenest en tant que Webview dans une application existante avec authentification SSO.

### Paramètre à activer

Activer **"Autoriser l'accès aux webviews"** dans le back-office.

### Effets

- Les utilisateurs accèdent à Serenest depuis une autre application sans ressaisir leurs identifiants.
- Cas d'usage typique : une application de restaurant ou d'entreprise qui intègre Serenest.

### Configuration

<Steps>
  <Step title="Activer le paramètre">
    Activer le paramètre dans le back-office.
  </Step>
  <Step title="Tester l'ouverture">
    Tester l'ouverture de Serenest depuis l'application hôte.
  </Step>
</Steps>

## Accès au Click & Collect

Deux types disponibles :

| Type | Description |
| --- | --- |
| **Natif** | Intégré directement dans Serenest |
| **PiClick** | Proposé par Pi Electronique |

### Paramètres à configurer

- Sélectionner le type (Natif ou PiClick)
- Saisir les URLs/APIs associées

### Étapes

<Steps>
  <Step title="Coordination technique">
    Coordonner avec les équipes techniques du client pour les APIs.
  </Step>
  <Step title="Configuration back-office">
    Configurer via le back-office.
  </Step>
  <Step title="Tests fonctionnels">
    Tester les parcours utilisateur (sélection, commande, paiement).
  </Step>
</Steps>
